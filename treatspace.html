Treatspace

Overview

  • Type: Healthcare Software Company in Pittsburgh, PA
  • Role: UX/UI Designer Internship
  • Tools: Sketch/Adobe Creative Suite/Zeplin
  • Timeline: 11 weeks, Summer 2019

In Summer 2019, I worked as a UX/UI intern at Treatspace. Treatspace is an award-winning, multi-faceted healthcare platform that enables high-performance referral management, powers online patient engagement, and facilitates clinical collaboration.

The core goal of the Treatspace platform, which is specifically designed for healthcare providers, has always has been to its customers (practices) succeed with a tailored specific experience.

I worked closely with engineers, other team members, but mostly my mentor Naveen Kumar, who guides me to finish design projects well and learn more about how to tackle design challenges, and how to design a complete end-to-end flow.

At the beginning of the intern, I still remember I spent some time understanding the Treatspace platform since I was new in the healthcare industry. But, as I got familiar with the platform, I got familiar with it and as well by attending the Dev daily/Team weekly meetings, It was clear that some product needed a re-designed of several components to make it more intuitive and usable.

During the project assignment, I was asked to choose one project out of 4 projects and was able to finish all of them in the end. Most of the validations of my ideas and designs were through meetings and design critique sessions.

I was able to handle and complete the following projects:

Project 1: A front-facing website editing feature (Responsive)

Project 2: New Internal Dashboard Design for the Treatspace Platform

Project 3: The PracticeBeat self-service Online V2 Testing Tool

Project 4: Illustrations on icons and presentation mockups

And, exported all designs into specs and guidelines for the engineering team.

Project 1: A front-facing website editing (Responsive)

OVERVIEW:

My first task was redesigning interface for Treatspace’s front-facing website editing feature using design thinking principles. It is a system enable users to customize or make changes to the experience to meet their specific needs by configuring layout, content, or system functionality.

COLLECT USE CASES:

After gathering the needs/requests from team, I was able to gather current use cases and existing data. After that, I started to find some inspiration from modern visual layouts. These layouts most likely aim for making layouts more accessible for users through a clearer UI, reducing clutter and improving user flow.

SKETCHING INITIAL IDEA:

After coming up with some design solutions, I started to sketch some of the initial ideas.

USER FLOW DIAGRAMS:

Created information architecture to provide positive user experience by planning and designing the information structure for websites.

ITERATIONG USER INTERFACE:

As moved on to designing the interface of responsive website, I wireframes all the user interface screens and conducted an initial user testing session.

FINAL VISUAL:

After internally conducting some A/B testing between some CTA buttons and layouts, I was able to refine the design from better to the best and came up with final high-fidelity design below which is much clearer in many aspects.

Necessary feedbacks were considred and changed accordingly based on the user data and were finally documented and handed-off to the developers.

Project 2: New Internal Dashboard

OVERVIEW:

How might we make the interface more simple and easy to read? Was the main goal of this project. I was able to redesigned interface of new internal dashboard for the Treatspace platform. This new internal dashboard is the index page of the control panel for a website’s content management system.

Project 3: The PracticeBeat Online V2 Testing Tool

OVERVIEW:

I created prototype of PracticeBeat self-service online v2 testing tool. It is a tool that analyzes the site’s front-end performance and offers optimization recommendations for the individual practice website. The existing online testing tool is robust and well-functional. However, the goal of redesigning this project was to improve user engagement and increase the user presences with highly visualized UI.

Project 4: Illustrations on icons and presentation mockups

OVERVIEW:

Created illustration of icon and mockups that goes into PracticeBeat Website.

    • Few Take-aways

    Expose your idea and designs:

    Get design critique and feedback from the team to move forward. Be humble and open-minded. It is a powerful way of getting the feedback and opportunity to unveil a few points that I have never thought of them.

    Define a Project Scopes to Test Success:

    For designs, it's not easy to judge if the proposed designs work well in the real world. Creating a framework and requirement documentation helps in exploring the problems in the right way and eventually leads to an impactful outcome.

    Empathy is a Best Friend:

    Empathy is one of the design processes that keep your design on the right track. Don't confuse users providing with too much information. Spend some time to understand the user's point of view and context. Be able to define every designed screen.

    Importance of Communication:

    Design is all about communication. Participating in the daily/weekly meetings and feedback sessions are the key for better team collaboration and design iteration.
